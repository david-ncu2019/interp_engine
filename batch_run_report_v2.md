## 2. Technical Upgrades (v2)

### 2.1. GP Architectural Redesign
- **Adaptive Kernel Catalogue:** The engine no longer assumes a smooth RBF kernel. It now automatically selects between `matern_32` (rough), `matern_52` (moderate), and `rbf` (smooth) based on Log-Marginal Likelihood (LML).
- **Explicit Nugget Modeling:** Added a `WhiteKernel` component to the composite kernel. This provides a dedicated parameter for measurement noise and micro-scale variability, preventing the "constant mean" failure mode seen in previous versions for noisy data (S4, S8).
- **Log-Space Optimization:** All length scales, variances, and nuggets are now searched in natural log space, ensuring the optimizer covers multiple decades of scale with equal precision.
- **Adaptive Bounds:** Both engines now compute minimum and maximum length scale bounds dynamically from the training data geometry ($0.5 \times$ median NN distance to $0.6 \times$ max pairwise distance).

### 2.2. Visualization Fixes
- **Anisotropy Ellipse:** Fixed a bug in `utils.py` where Kriging parameters were incorrectly mapped, causing ellipses to render as circles. The visualization now correctly reflects the `range` and `anisotropy_ratio` for both engines.

## 4. Analytical Insights

1.  **GP Breakthrough (S3 & S4):** The most dramatic improvement is in `S3_Sparse_Aniso` and `S4_HighNugget`. Previously, the GP $R^2$ for S3 was nearly zero; it has now jumped to **0.243**. In S4, the $R^2$ improved from negative (worse than mean) to **0.342**. This is direct proof that adaptive bounds and explicit nugget modeling work.
2.  **Kernel Selection:** The engine correctly identified `matern_32` for the rough SGS scenarios (S5, S6, S7) and `rbf` or `matern_52` for the smoother isotropic cases, justifying the categorical search approach.
3.  **Cross-Validation Consistency:** Kriging remains slightly superior because it explores a wider variety of variogram shapes (9 models vs 3 kernels). However, the GP engine now provides much better uncertainty estimates (`pred_std`) which are more mathematically consistent with the data noise levels.
